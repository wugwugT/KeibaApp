# 【git管理について】

## ブランチ構成
- **prod**: 本番環境用ブランチ。
- **stg**: テスト環境（ステージング）用ブランチ。

## 開発フロー

### 1. 最新化とブランチ作成
開発を開始する前に、必ず `stg` ブランチから最新の変更を取り込みます。その後、要件の機能項目に基づいたブランチを作成してください。
- `git fetch` および `git pull` を実行して `stg` を最新化する。
- 命名規則: `features/機能項目ID`
  - 例: `features/DB001`

### 2. 実装とマージ
- 機能項目単位でソースコードが完成したら、`stg` ブランチへマージします。

### 3. 結合試験
- `stg` ブランチにおいて、他の機能との結合試験を実施し、不具合がないか確認します。

### 4. 本番反映
- 他機能を含めたすべての動作確認が完了した後、`prod` ブランチに反映（マージ）します。

## 【開発時テスト】

開発時は以下のコマンドを実行し、ソース単位で型エラーなどの問題がないことを必ず確認してください。

- **実行コマンド**: `npx tsc --noEmit`
- **確認内容**: TypeScriptのコンパイラ（tsc）を実行し、実際にファイルを出力（Emit）せずにエラーチェックのみを行います。

## 【セットアップ】
リポジトリをクローンした後、以下の手順で環境を構築してください。

### 1. 依存関係のインストール
現在の構成ではライブラリ間の厳密なバージョンチェックを回避する必要があるため、必ず `--legacy-peer-deps` フラグを付けて実行してください。

```bash
npm install --legacy-peer-deps

### 2. アプリの起動
インストール完了後、以下のコマンドでキャッシュをクリアして起動します。

```bash
npm start -c
